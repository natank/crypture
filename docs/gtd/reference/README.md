# Reference

**Purpose**: Non-actionable but useful information to keep accessible.

## Organization

Create subfolders by category:
- `business/` - Business plans, strategies, market research
- `technical/` - Technical documentation, APIs, tools
- `marketing/` - Marketing materials, competitor analysis
- `partnerships/` - Partnership information, contacts
- `legal/` - Legal documents, terms, policies
- `inspiration/` - Ideas, examples, case studies

## File Naming

`{category}/{title}.md`

## When to Use Reference

- **Informational**: Useful but no action required
- **Future Use**: Might need later but not now
- **Completed Projects**: Archive of finished work
- **Resources**: Tools, links, contacts

## Template

```markdown
# {Title}
**Category**: {business/technical/marketing/partnerships/legal/inspiration}
**Added**: {YYYY-MM-DD}
**Tags**: {tag1, tag2, tag3}

## Summary
{Brief description of what this is and why it's useful}

## Content
{The actual reference information}

## Related Items
- {Links to related projects, actions, or other reference items}
```

## Maintenance

- Review monthly during weekly review
- Remove outdated or no longer useful items
- Archive completed project materials here
